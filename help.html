Help & FAQ

General
What is SecureDNS Guard? +
SecureDNS Guard is a DNS-level security proxy. It intercepts outgoing DNS requests, runs them through a Machine Learning model (Random Forest) to detect threats like Domain Generation Algorithms (DGA) and DNS Tunneling, and blocks malicious queries in real time. Safe queries are forwarded securely via DNS-over-HTTPS.
How does the ML detection work? +
For every DNS query the engine extracts 10 lexical features (entropy, domain length, digit ratio, etc.) and feeds them into a pre-trained Random Forest classifier. If the predicted malicious confidence exceeds the configured threshold, the query is blocked with an NXDOMAIN response.
What is DNS-over-HTTPS (DoH)? +
DoH encrypts your DNS queries inside HTTPS so that your ISP or anyone on the network cannot see which domains you visit. SecureDNS Guard forwards all "safe" queries through DoH providers like Cloudflare, Google, or Quad9.
Configuration
How do I change the detection sensitivity? +
Go to Settings and adjust the blocking threshold slider. A lower value (e.g. 70%) blocks more aggressively while a higher value (e.g. 95%) is more conservative, allowing more queries through.
Where can I see blocked queries? +
The Query Log page shows every DNS query processed by SecureDNS Guard. You can filter by verdict (blocked/safe), attack type (DGA/tunneling), confidence level, and date range. Click "Explain" on any row to see the ML features.
API & Integration
How do I use the API? +
Generate an API key on the API Keys page, then send POST requests to /api/v1/check with your key in the X-API-Key header. Full details and code examples are on the Setup Guide page.
What are the rate limits? +
Free plan: 100 API queries per day with 1 key. Pro plan: 10,000 queries per day with up to 5 keys. Rate limits reset daily at midnight UTC. The admin can upgrade plans from the admin panel.
How do I set up the DNS proxy on a client? +
Go to your Profile page and click "Download dns_proxy.py". The script comes pre-configured with your API key. Run it with administrator/root privileges on the client machine: python dns_proxy.py. It will intercept DNS queries on port 53.
