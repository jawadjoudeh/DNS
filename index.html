How it works

Three steps to protection

From raw DNS query to final verdict in milliseconds — here's what happens under the hood.

01
Query Interception
A lightweight Python proxy listens on UDP port 53, capturing every DNS query from your machine before it reaches any upstream resolver — zero configuration required.
02
ML Classification
10 lexical features — Shannon entropy, domain length, digit ratio, subdomain depth, TLD risk, and more — are extracted and scored by the IsolationForest anomaly detector in under 5 ms. DoH network flows use a separate RandomForest classifier.
03
Block or Forward via DoH
Threats are blocked and logged with a feature breakdown. Safe domains are forwarded encrypted via DNS-over-HTTPS to Cloudflare, Google, or Quad9 — your ISP sees nothing.
Capabilities

Everything you need

Enterprise-grade DNS security built as a graduation project at the University of Jordan – KASIT.

Dual ML Pipeline
An IsolationForest anomaly detector scores 10 lexical features per domain. A separate RandomForest classifier analyzes 29 DoH network-flow features — two independent layers of detection.
DNS-over-HTTPS
All safe traffic is forwarded encrypted via DoH to Cloudflare, Google, or Quad9. Your ISP, router, and any network observer see zero DNS queries.
Live Dashboard
Monitor every query in real time, inspect each blocked domain's ML feature breakdown, and track model accuracy, precision, recall, and F1 over time.
REST API
Classify single domains or batches of up to 50 via API key authentication. Every response includes a confidence score and the top feature contributions.
Multi-User & Roles
Full admin panel for managing users, reviewing all logs, triggering model retraining, and auditing system activity — isolated from the standard user view.
Continuous Retraining
Admins trigger on-demand model retraining directly from the dashboard. Accuracy, precision, recall, and F1 are recorded for each model version.
